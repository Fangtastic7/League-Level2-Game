Whack A Mole!!!

by: Max Fang

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I decided to make this game because I was first intrigued by the first Whack-a-mole game in Level 1. The idea of smashing moles gave me excitement, when I have the ability to spam mouse one because it does require too much effort. In addition, I am a fan of intense games, where there is factors putting pressure on the user.

2. What I Learned

From user testing, I learned that there were glitches with the hammer. From my classmates, they explained one glitch, where the hammer would not follow the mouse when the cursor was dragged along. Also, I was advised that I should fix the borders, so the hammer would not go off the screen and that there should be an accuracy indicator.

3. Architecture of My Game

Whack-a-mole has several different classes that correspond and communicate with each other. When the game first starts, the starter class first creates the frame and panel of the game. After, the GamePanel creates the visual components of the screen, excluding the hammer and moles. The hammer and moles have their own separate classes, where each of them is drawn. In order to manage the location of each mole, a object manager class is created in order to randomize the location of the moles for different intervals of time. For creating the timer and music, separate classes were also used to call on to methods. Overall, the main central class was the GamePanel class, which it connected all the classes together.

4. Technical Implementation

When I began, I first created my starter class and GamePanel class. After these classes were created, I made other classes for other components such as the Hammer, Moles, etc. The particular aspect of the game that I am most proud of was the animation of the hammer. In order to make the animation, I had to use two pictures and code from an animation demo. In addition, I had to track the cursor's location and match it with the hammer, so that it would stay with the cursor.

5. Most Challenging Part

The most challenging part of this project was making the speed of the intervals between the changing locations of the moles. At first, I thought that changing the speed would be simple and all I had to do was set the interval value of the mole-timer to a specific value. However, it did not turn out right. I debugged and tested the intervals, and it did not work. In order to resolve this problem, I had to go online and search for answers. As a result, I did find that the swing timer had a function, where you could specifically change the delay.

6. Future Fixes and Updates

Some improvements or fixes that I would like to make in the future include making more holes on the screen and creating different skins for the hammer itself. I were to continue working on this game, I would improve it by creating more holes and classes, so that the game can change dynamically. This way, the game play may be more thrilling.
